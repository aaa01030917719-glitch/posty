# Posty Share Approval Policy

## Purpose

Posty의 공유/컨펌 기능은 외부 사람이 콘텐츠를 확인할 수 있게 하기 위한 기능입니다.

대표, 클라이언트, 협업자는 Posty에 로그인하지 않아도 공유받은 콘텐츠를 읽기 전용으로 확인할 수 있어야 합니다. 공유받은 사람에게 수정 권한은 제공하지 않습니다.

이 문서는 공유 URL, 토큰, DB schema, route, UI를 구현하기 전에 제품 정책과 후속 설계 방향을 정리하기 위한 문서입니다.

## Share Scope

1차 공유 단위는 콘텐츠 카드 1개입니다.

공유 대상에 포함할 수 있는 정보:

- 콘텐츠 카드 1개
- 콘텐츠 원고
- 대본/씬
- 캡션
- 해시태그
- 썸네일 문구
- 체크리스트
- 업로드 예정일
- 캠페인명
- 채널 정보

## Excluded Information

공유 화면에는 내부 작업 정보와 시스템 식별자를 노출하지 않습니다.

공유하지 않는 정보:

- 내부 메모
- activity log 전체
- 삭제/복구 이력
- 사용자 계정 정보
- Supabase 내부 id
- 편집용 상태 정보 중 외부 확인에 불필요한 정보

## Share Unit Policy

- 1차 공유 단위는 `content_cards`의 콘텐츠 카드 1개입니다.
- 캠페인 전체 공유는 후속 확장으로 분리합니다.
- 월간/주간 캘린더 공유도 후속 확장으로 분리합니다.
- 공유 화면은 Posty의 중심 작업 공간인 `/content/[id]`를 대체하지 않습니다.
- 실제 편집과 저장은 계속 Posty 내부의 콘텐츠 상세 화면에서만 수행합니다.

## Permission Policy

- 공유 링크는 읽기 전용입니다.
- 공유받은 사람은 수정, 삭제, 복구, 상태 변경을 할 수 없습니다.
- 로그인 없이 접근 가능한 공개 토큰 기반 URL을 후속 작업에서 고려합니다.
- 공유 링크는 비활성화할 수 있어야 합니다.
- 공유 링크가 비활성화된 콘텐츠는 외부 공유 화면에서 접근할 수 없어야 합니다.

## Approval Policy

1차에서는 댓글 기능을 만들지 않습니다.

후속 컨펌 상태는 아래 정도로 확장할 수 있습니다.

- 검토중
- 승인
- 수정 요청

이번 단계에서는 공유/컨펌 정책만 문서화합니다. 승인/반려 기능, 댓글, 알림, 권한 관리는 구현하지 않습니다.

## Security Policy

- 공유 token은 추측하기 어려운 값이어야 합니다.
- 삭제된 콘텐츠는 공유 링크에서도 접근할 수 없어야 합니다.
- 공유 비활성화된 콘텐츠는 접근할 수 없어야 합니다.
- 내부 작성자 정보와 private memo는 노출하지 않습니다.
- 공유 응답은 외부 확인에 필요한 최소 필드만 포함해야 합니다.
- 공개 공유 화면은 편집 API를 호출할 수 없어야 합니다.

## Follow-Up DB Draft

이번 작업에서 SQL은 만들지 않습니다. 아래는 후속 DB 설계 초안입니다.

예상 테이블:

- `content_share_links`

예상 필드:

- `id`
- `user_id`
- `card_id`
- `token`
- `is_enabled`
- `expires_at`
- `created_at`
- `disabled_at`

초안 정책:

- `card_id`는 콘텐츠 카드 1개를 가리킵니다.
- `token`은 공개 공유 URL의 접근 키로 사용합니다.
- `is_enabled = false`이면 공유 페이지 접근을 차단합니다.
- `expires_at`이 지난 링크는 접근을 차단합니다.
- 삭제된 콘텐츠는 링크가 활성 상태여도 접근을 차단합니다.

## Follow-Up Route Draft

이번 작업에서 route는 만들지 않습니다. 아래는 후속 route 초안입니다.

예상 route:

- `/share/content/[token]`

후속 route 정책:

- token으로 공유 링크를 조회합니다.
- 공유 링크가 없거나 비활성화되었거나 만료되었으면 접근을 차단합니다.
- 연결된 콘텐츠 카드가 삭제된 상태이면 접근을 차단합니다.
- 공유 화면은 읽기 전용으로 렌더링합니다.
- 내부 id, 계정 정보, 내부 메모, 전체 activity log는 응답에 포함하지 않습니다.

## Not Implemented In This Step

이번 작업에서는 아래 항목을 구현하지 않습니다.

- 공유 링크 생성 버튼
- 공유 공개 페이지
- DB schema
- SQL 파일
- 댓글
- 승인/반려 기능
- 알림
- 권한 관리
- 팀 협업
- route 생성
- UI 구현

## Summary

Posty의 공유/컨펌 기능은 콘텐츠 카드 1개를 외부 사람이 읽기 전용으로 확인하는 흐름에서 시작합니다. 편집과 저장은 내부 콘텐츠 상세 화면에 남겨두고, 공유 화면은 외부 확인에 필요한 정보만 최소로 노출합니다. 삭제된 콘텐츠, 비활성화된 링크, 만료된 링크는 접근할 수 없어야 합니다.
