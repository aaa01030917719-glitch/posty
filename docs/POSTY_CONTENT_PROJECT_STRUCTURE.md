# POSTY_CONTENT_PROJECT_STRUCTURE.md

이 문서는 구현 문서가 아니라, Posty의 콘텐츠 프로젝트 상위 구조 정책을 정리한 초안입니다.
앱 코드, DB schema, SQL 적용 범위는 포함하지 않으며 후속 작업 전에 사용자 확인이 필요합니다.

## 1. 콘텐츠 프로젝트의 역할

Posty에는 콘텐츠 카드보다 상위 개념인 콘텐츠 프로젝트가 필요합니다.

콘텐츠 프로젝트는 하나의 주제, 목표, 캠페인 아래에서
여러 채널 콘텐츠를 묶는 대분류입니다.

예:
- 콘텐츠 프로젝트: Posty 콘텐츠 프로젝트
- 하위 콘텐츠:
  - 인스타 카드뉴스
  - 스레드
  - 유튜브 쇼츠
  - 네이버TV
  - 블로그 글

즉, 채널은 발행 위치이고,
프로젝트는 여러 채널 콘텐츠를 묶는 주제 단위입니다.

## 2. 콘텐츠 카드와의 관계

정책:
- 콘텐츠 프로젝트는 콘텐츠 카드의 상위 묶음입니다.
- 콘텐츠 카드는 0개 또는 1개의 콘텐츠 프로젝트에 연결될 수 있습니다.
- 하나의 콘텐츠 프로젝트 아래에는 여러 콘텐츠 카드가 들어갈 수 있습니다.
- 콘텐츠 카드가 프로젝트에 속하지 않는 상태도 허용합니다.

중요:
- 채널은 발행 위치입니다.
- 프로젝트는 주제 묶음입니다.
- format은 콘텐츠 형식입니다.
- 이 세 개념을 서로 혼동하지 않습니다.

## 3. 데이터 구조 정책

현재 정책 기준:
- `project_id`는 `content_cards`에 둡니다.
- 콘텐츠 프로젝트 자체의 실제 테이블명, SQL, migration 구조는 이번 단계에서 확정하지 않습니다.
- 이번 문서는 제품 구조 정책만 정의합니다.

즉, 현재 단계에서는 아래까지만 확정합니다.
- 콘텐츠 카드가 프로젝트를 가리키는 단일 참조를 가진다.
- 프로젝트는 카드의 상위 분류 역할을 한다.

## 4. 화면 표시 정책

프로젝트명은 아래 위치에 표시합니다.

- 콘텐츠 목록
- 일정 카드
- 콘텐츠 상세 breadcrumb
- 콘텐츠 상세 메타 정보

표시 목적:
- 같은 주제 아래 묶인 여러 채널 콘텐츠를 빠르게 식별하기 위함
- 일정 화면과 목록 화면에서 카드의 맥락을 바로 이해하기 위함

## 5. 캠페인 목록 UI 정책

Posty에서 콘텐츠 프로젝트는 사용자 화면에서 "캠페인"으로 표현합니다.

캠페인 목록은 카드형 UI가 아니라 리스트형 row UI를 기본으로 합니다.

이유:
- 캠페인은 콘텐츠를 담는 큰 묶음이지만, 콘텐츠 자체는 아닙니다.
- 캠페인마다 큰 카드 박스를 만들면 화면이 무거워지고, 콘텐츠 관리 앱보다 프로젝트 관리툴처럼 보일 수 있습니다.
- Posty의 콘텐츠 페이지는 가볍고 빠르게 훑어볼 수 있어야 하므로, 캠페인은 행 단위로 정리합니다.

기본 캠페인 row 구조:
- 펼침/접힘 아이콘
- 캠페인 제목
- 캠페인에 포함된 콘텐츠의 채널 요약 뱃지

예:
```text
⌄  Posty 콘텐츠 프로젝트                         인스타그램  스레드
```

캠페인 안에 콘텐츠가 없으면 채널 뱃지는 표시하지 않습니다.

예:
```text
⌄  Posty 콘텐츠 프로젝트
```

채널 요약 뱃지 정책:
- 캠페인 row 오른쪽의 채널 뱃지는 캠페인 자체의 성격이 아닙니다.
- 해당 캠페인 안에 실제로 작성된 콘텐츠들의 채널 요약입니다.
- 예를 들어 캠페인 안에 인스타그램 콘텐츠와 스레드 콘텐츠가 있으면 인스타그램, 스레드 뱃지를 표시합니다.
- 같은 채널 콘텐츠가 여러 개 있어도 채널 뱃지는 1번만 표시합니다.

아이디어 표시 정책:
- 캠페인 row에는 `아이디어` 상태 뱃지를 표시하지 않습니다.
- 아이디어는 별도 아이디어 게시판에서 관리합니다.
- 캠페인 목록에서 아이디어 표시를 반복하면 역할이 겹치므로 제외합니다.

상태 카운트 정책:
- 캠페인 row에는 기본적으로 상태 카운트를 표시하지 않습니다.
- 예: `임시저장 2`, `완료 1`, `기획중 1`
- 위와 같은 상태 카운트는 기본 row에서는 생략합니다.
- 상태는 펼친 하위 콘텐츠 row에서 개별 콘텐츠 단위로 표시합니다.
- 필요하다면 후속 작업에서 캠페인 상세 또는 별도 요약 영역으로 분리합니다.

펼친 상태의 하위 콘텐츠 row 예:
```text
⌄  Posty 콘텐츠 프로젝트                         인스타그램  스레드

   인스타 카드뉴스 0504                          인스타그램   임시저장
   스레드 글 0504                                스레드       완료
```

접힌 상태 예:
```text
›  Posty 콘텐츠 프로젝트                         인스타그램  스레드
```

콘텐츠가 없는 캠페인 예:
```text
⌄  Posty 콘텐츠 프로젝트
```

중요 디자인 정책:
- 캠페인 row를 큰 card/box로 감싸지 않습니다.
- 의미 없는 rounded border wrapper를 만들지 않습니다.
- 리스트 행은 여백, 정렬, hover 정도로 구분합니다.
- 실제 콘텐츠 카드, 패널, 모달, empty state처럼 독립 의미가 있는 요소에만 card를 사용합니다.
- 이 정책은 `design-system.md`의 "의미 없는 wrapper box/card 금지" 원칙과 일치해야 합니다.

이번 문서에서 구현하지 않는 것:
- `content_projects` 테이블 생성
- `content_cards.project_id` 추가
- 캠페인 목록 UI 구현
- 캠페인 펼침/접힘 상태 구현
- 채널 요약 뱃지 계산 구현
- 캠페인 상세 페이지 구현
- DB SQL 작성

## 6. 후속 작업으로 미루는 범위

이번 정책 문서에서 구현하거나 확정하지 않는 것:

- 프로젝트 상세 페이지
- 프로젝트 생성/수정 UI
- 프로젝트 목록 페이지
- `content_projects` 실제 테이블 SQL
- Supabase migration
- 프로젝트 기반 필터/정렬 UI
- 프로젝트 권한 정책

프로젝트 상세 페이지는 후속 작업으로 분리합니다.

## 7. 요약

Posty의 구조는 아래처럼 정리합니다.

- 프로젝트: 주제/목표/캠페인 단위
- 콘텐츠 카드: 실제 발행 단위
- 채널: 발행 위치
- format: 콘텐츠 형식

핵심 관계:
- 프로젝트 1개 아래 여러 콘텐츠 카드가 올 수 있습니다.
- 콘텐츠 카드 1개는 프로젝트 0개 또는 1개에 연결됩니다.
