# POSTY_PRODUCT_STRUCTURE.md

## 1. Posty 서비스 목적

Posty는 단순 캘린더 앱이 아닙니다.

인스타그램, 스레드, 유튜브, 블로그 등의 콘텐츠 업로드 일정, 원고, 아이디어, 마인드맵을 한 흐름으로 관리하는 콘텐츠 운영용 업무 웹앱입니다.

핵심 흐름:

아이디어 → 콘텐츠 카드 → 원고/대본 → 일정 배치 → 발행 관리

## 2. 중심 객체 정의

Posty의 중심 객체는 콘텐츠 카드입니다.

콘텐츠 카드 1개는 예를 들어 아래처럼 정의됩니다.

예:
- 인스타 릴스 0504
- 유튜브 쇼츠 0504
- 스레드 콘텐츠 0504
- 블로그 5월 2주차

콘텐츠 카드 안에 포함될 수 있는 정보:
- 제목
- 채널
- 콘텐츠 형식
- 상태
- 업로드 예정일
- 원고
- 대본/씬
- 캡션
- 해시태그
- 썸네일 문구
- 체크리스트
- 메모
- 참고 링크
- 아이디어 출처
- 마인드맵 연결
- 작업 이력
- 공유/컨펌 상태

중요:
원고, 대본, 캡션, 해시태그, 썸네일 문구, 체크리스트, 메모, 참고 링크는 독립된 최상위 객체라기보다 콘텐츠 상세 안의 작업 섹션으로 봅니다.

## 3. 콘텐츠 형식별 섹션 정책

모든 콘텐츠 형식에 모든 섹션이 필요한 것은 아닙니다.

따라서 콘텐츠 상세는 아래 정책을 따릅니다.

- 콘텐츠 상세는 통합 작업 공간입니다.
- 콘텐츠 형식에 따라 기본 노출 섹션이 달라집니다.
- 필요 없는 섹션은 숨김 또는 접힘 상태로 관리합니다.
- 사용자는 필요한 섹션을 추가할 수 있습니다.
- 입력하지 않은 섹션을 항상 빈 필드로 길게 노출하지 않습니다.

형식별 기본 섹션 예시:

### 3.1 유튜브 쇼츠 / 인스타 릴스
- 개요
- 대본/씬
- 캡션
- 해시태그
- 썸네일 문구
- 체크리스트
- 일정
- 작업 이력

### 3.2 스레드
- 개요
- 본문
- 후킹 문장
- 키워드
- 일정
- 작업 이력

### 3.3 블로그
- 개요
- 본문
- 목차
- 키워드
- 참고 링크
- CTA
- 일정
- 작업 이력

### 3.4 카드뉴스
- 개요
- 슬라이드별 문구
- 캡션
- 디자인 체크리스트
- 참고 링크
- 일정
- 작업 이력

추가 가능한 섹션 예시:
- 대본/씬
- 캡션
- 해시태그
- 썸네일 문구
- SEO 키워드
- 참고 링크
- 체크리스트
- 컨펌 메모
- 작업 이력

## 4. 라우팅 구조 정책

추천 라우팅 구조:

### /schedule
- Posty의 메인 화면
- 기본값은 월간 캘린더가 아니라 주간 타임라인 캘린더
- 왼쪽에는 오늘 할 일 / 작업 리스트 패널
- 오른쪽에는 주간 콘텐츠 일정 타임라인
- 월/주/일 전환은 유지
- 일정 카드 클릭 시 콘텐츠 상세로 이동하거나 상세 패널을 엽니다.

### /content
- 콘텐츠 카드 전체 관리 화면
- 카드뷰 / 리스트뷰 전환
- 검색 / 필터 / 정렬
- 새 콘텐츠 생성

### /content/[id]
- 콘텐츠 상세 작업 공간
- 원고, 대본, 캡션, 일정, 체크리스트, 메모, 링크, 작업 이력을 모두 여기에서 관리
- Posty의 핵심 상세 화면

### /scripts
- 독립 원고 앱이 아니라 원고 작업이 있는 콘텐츠 모아보기 화면
- 작성 중 원고, 최종본 대기, 캡션 미작성, 대본 있음 등의 필터 중심
- 실제 편집은 /content/[id] 내부에서 진행

### /ideas
- 아이디어 관리
- 아이디어를 콘텐츠 카드로 전환

### /mindmap
- 마인드맵 관리
- 노드를 아이디어 또는 콘텐츠 카드로 전환/연결

### /timeline
- 작업 이력 전체 페이지
- 임시저장, 작성완료, 삭제됨 같은 activity log 확인

### /dashboard
- 전체 운영 현황 요약
- 우선순위는 /schedule보다 낮음

## 5. /schedule 메인 화면 정의

/schedule은 Posty의 첫 화면입니다.

기본값:
- 주간 타임라인 뷰

구성:
1. 미니멀 사이드바
2. 오늘 할 일 / 작업 리스트 패널
3. 주간 콘텐츠 일정 타임라인
4. 상단 액션 영역

Page header:
- 제목: 콘텐츠 일정
- 설명: 이번 주 콘텐츠 업로드 흐름을 확인하고 관리하세요
- 우측 액션:
  - 새 콘텐츠
  - 타임라인 아이콘
  - 추후 공유 버튼 위치

Today panel:
- 오늘 할 일
- 임시저장 글
- 작성완료 대기
- 컨펌 필요
- 이번 주 주요 작업

Calendar workspace:
- 주간 타임라인 캘린더
- 날짜/시간대
- 콘텐츠 일정 카드
- 채널 표시
- 상태 badge
- 원고/대본 진행 상태

View switch:
- 월 / 주 / 일
- 기본 선택값: 주

## 6. 타임라인 용어 구분

Posty에서 타임라인은 두 가지 의미가 있을 수 있으므로 명확히 구분합니다.

### 6.1 주간 타임라인 캘린더
- /schedule의 메인 캘린더 뷰
- 날짜/시간대별 콘텐츠 발행 일정을 보여줌

### 6.2 작업 이력 타임라인
- /timeline 또는 TimelineDrawer에서 사용
- 임시저장, 작성완료, 삭제됨 같은 최근 작업 기록을 보여줌

두 개를 혼동하지 않습니다.

## 7. 작업 이력 타임라인 정책

목적:
여러 콘텐츠를 동시에 작업할 때, 내가 어디까지 완료했는지 빠르게 확인하기 위함입니다.

표시 기준 MVP:
- 임시저장
- 글작성완료
- 삭제됨

항목 정보:
- 콘텐츠 제목
- 상태
- 최초 작성일
- 활동 시간

예:
- 지피티 스레드 콘텐츠 0504
- 상태: 임시저장
- 작성일: 26.05.04
- 활동 시간: 26.05.04 14:43

/timeline:
- 작업 이력 전체 페이지
- 최신순 리스트
- 상태 필터:
  - 전체
  - 임시저장
  - 완료
  - 삭제됨
- 항목 클릭 시 해당 콘텐츠 상세로 이동

TimelineDrawer:
- /schedule 우측 상단 아이콘 클릭 시 열림
- 오른쪽에서 슬라이드 인
- 최근 작업 이력 10~20개 표시
- 전체 타임라인 보기 클릭 시 /timeline 이동
- 메인 캘린더보다 우선순위가 높아 보이면 안 됨

## 8. 콘텐츠 상세 날짜 표시 정책

작성일:
- 최초 생성일 기준
- 고정값

수정일:
- 마지막 수정 / 저장 / 상태 변경 시간 기준

표시 예:
작성일 26.05.04 · 마지막 수정 26.05.04 14:43

삭제 이력을 보여주려면:
- hard delete보다 soft delete 또는 activity log 구조를 고려합니다.

## 9. 외부 공유 기능 정책

지금 당장 구현 대상은 아닙니다.

하지만 추후 대표/클라이언트 컨펌을 위해 비회원에게 읽기 전용 URL 공유가 필요합니다.

공유 가능 대상:
- 콘텐츠 카드
- 원고/대본
- 월간/주간 일정
- 콘텐츠 캘린더

정책:
- 공유받은 사람은 로그인하지 않아도 볼 수 있음
- 수정은 불가능
- 읽기 전용
- 링크 끄기/만료일 설정은 추후 고려

메인 화면에서는 지금 당장 구현하지 않고, 추후 공유 버튼이 들어갈 위치만 고려합니다.

## 10. 디자인 방향 메모

- Airbnb md color palette 유지
- primary `#ff385c`는 CTA, active, 오늘 날짜, 선택 상태에만 제한적으로 사용
- 전체 화면을 핑크로 물들이지 않음
- 텍스트는 ink/body 중심으로 또렷하게
- muted는 보조 정보에만 사용
- 의미 없는 얇은 구분선, wrapper border, divider 남발 금지
- 정보 구분은 여백, 배경 톤, 카드 구조를 우선 사용
- 사이드바는 미니멀하고 사용성 중심
- 귀엽고 큰 UI보다 글자가 잘 보이는 업무용 SaaS 느낌
- 콘텐츠 카드 왼쪽에 의미 없는 세로 포인트 선을 넣지 않음
- 채널/상태는 작은 badge 또는 텍스트로 구분
- 카드 전체 색상 사용은 절제

## 11. 이번 문서에서 구현하지 않는 것

이번 작업에서는 아래를 절대 구현하지 않습니다.

- 실제 /timeline route 생성
- TimelineDrawer 컴포넌트 생성
- /schedule UI 수정
- /content/[id] route 생성
- DB schema 수정
- activity log 테이블 생성
- 공유 기능 구현
- 페이지 리디자인
- 기존 기능 수정



## 12. 주간 뷰 날짜 표시 정책

/schedule의 주간 타임라인 뷰 상단 제목은 날짜 범위만 표시하지 않고,
월별 주차명을 중심으로 표시합니다.

기본 표시:
- 2026년 5월 1주차

보조 텍스트:
- 5월 3일 - 5월 9일

주차 계산 MVP 기준:
- 해당 월의 1일이 포함된 주를 1주차로 봅니다.
- 그다음 주를 2주차, 3주차로 계산합니다.
- 주간 뷰는 일요일 시작 기준을 사용합니다.
- 월이 걸치는 주는 현재 기준 월의 주차명으로 표시하고,
  정확한 날짜 범위는 보조 텍스트로 함께 표시합니다.

예:
- 2026년 5월 1주차
- 4월 28일 - 5월 4일
